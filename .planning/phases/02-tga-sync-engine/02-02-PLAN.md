    **1. Hash computation:**
    ```typescript
    import { createHash } from 'crypto';

    private computeQualificationHash(tgaData: TgaQualificationDetail): string {
      const fields = {
        title: tgaData.title,
        status: tgaData.status,
        supersededBy: tgaData.supersededBy,
        trainingPackage: tgaData.trainingPackage?.code ?? null,
      };
      return createHash('sha256').update(JSON.stringify(fields)).digest('hex');
    }

    private computeUnitHash(tgaData: TgaUnitDetail): string {
      const fields = {
        title: tgaData.title,
        status: tgaData.status,
        supersededBy: tgaData.supersededBy,
        elements: tgaData.elements.map(e => ({
          num: e.num,
          title: e.title,
          pcs: e.performanceCriteria.map(pc => ({ num: pc.num, text: pc.text })),
        })),
      };
      return createHash('sha256').update(JSON.stringify(fields)).digest('hex');
    }
    ```

    **2. Alert deduplication helper:**
    ```typescript
    private async createAlertIfNotExists(data: {
      rto_id: string;
      alert_type: string;
      severity: string;
      entity_type: string;
      entity_id: string;
      title: string;
      description: string;
    }): Promise<boolean> {
      // Returns true if alert was created, false if it already existed
      const existing = await prisma.complianceAlert.findFirst({
        where: {
          rto_id: data.rto_id,
          alert_type: data.alert_type,
          entity_id: data.entity_id,
          resolved: false,
          deleted_at: null,
        },
      });
      if (existing) return false;
      await prisma.complianceAlert.create({ data: { ...data, resolved: false } });
      return true;
    }
    ```

    **3. RTO status_color update:**
    ```typescript
    private async updateRtoStatusColor(rtoId: string): Promise<void> {
      const alerts = await prisma.complianceAlert.findMany({
        where: { rto_id: rtoId, resolved: false, deleted_at: null },
        select: { severity: true },
      });
      let color = 'green';
      if (alerts.some(a => a.severity === 'red')) color = 'red';
      else if (alerts.some(a => a.severity === 'amber')) color = 'amber';
      await prisma.rto.update({ where: { id: rtoId }, data: { status_color: color } });
    }
    ```

    **4. syncQualification(qualCode: string, syncLogId: string): Promise<{ changed: boolean; alertsCreated: number }>**
    Logic:
    a. Fetch TGA data: `await this.tgaApiClient.getQualificationDetail(qualCode)` — if null (not found), log warning and return { changed: false, alertsCreated: 0 }
    b. Check if Qualification exists in DB: `prisma.qualification.findUnique({ where: { code: qualCode } })`
    c. Compute hash of TGA data
    d. If no DB record: create new Qualification (upsert). No snapshot needed — first time seen.
    e. If DB record exists AND hash matches: no change — return { changed: false, alertsCreated: 0 }
    f. If hash differs:
       - Create QualificationSnapshot with snapshot_data = (full previous DB record as JSON), change_fields = (list of fields that changed by comparing old vs new values)
       - Update Qualification record with new TGA data and new hash, updated last_synced_at
       - Flag affected RTOs: find all RtoQualification rows with qualification_id = this qual's id and deleted_at null
       - For each affected RTO:
         - If new status is "Superseded": createAlertIfNotExists with alert_type "QUAL_SUPERSEDED", severity "red", entity_type "qualification", entity_id = qual.id, title = "Qualification {code} has been superseded by {supersededBy ?? 'unknown'}", description = "TGA has marked this qualification as superseded."
         - If title/wording changed (not just status): createAlertIfNotExists with alert_type "QUAL_WORDING_CHANGED", severity "amber", entity_type "qualification", entity_id = qual.id, title = "Qualification {code} wording updated", description = "Training.gov.au has updated the qualification wording."
         - Call updateRtoStatusColor(rto.rto_id) after creating alerts
    g. Return { changed: true, alertsCreated: count }

    **5. syncUnit(unitCode: string, syncLogId: string): Promise<{ changed: boolean; alertsCreated: number }>**
    Logic mirrors syncQualification but for units:
    a. Fetch TGA unit detail
    b. Upsert Unit record (create if new, compare hash if existing)
    c. Upsert UnitElements and PerformanceCriteria: delete old elements for this unit, insert fresh from TGA response (simpler than diffing elements individually)
    d. If hash differs:
       - Create UnitSnapshot with full previous state as snapshot_data (include old elements from DB before deleting)
       - Update Unit record with new hash
       - Flag affected RTOs: find all QualificationUnit rows for this unit, then find all RtoQualification rows for those qualifications
         - Deduplicate by rto_id (same RTO may have multiple quals with this unit)
       - For each affected RTO: createAlertIfNotExists with alert_type "UNIT_WORDING_CHANGED", severity "amber", entity_type "unit", entity_id = unit.id, title = "Unit {code} content updated", description = "TGA has updated the element or performance criteria wording for this unit."
       - Call updateRtoStatusColor for each affected RTO

    **6. syncAll(triggeredBy: string): Promise<SyncResult>**
    This is the entry point called by the cron scheduler:
    a. Create SyncLog with status "running"
    b. Fetch all Qualification codes currently in DB: `prisma.qualification.findMany({ select: { code: true } })`
       - Also sync qualifications linked to any RTO via rto_qualifications (they may not be in the global qualifications table yet)
    c. For each qual code: call syncQualification(), accumulate counts, add 100ms delay between requests
    d. Fetch all Unit codes from DB: `prisma.unit.findMany({ select: { code: true } })`
    e. For each unit code: call syncUnit(), accumulate counts, add 100ms delay
    f. Update SyncLog with status "completed", all counts, completed_at = now()
    g. Return SyncResult with totals

    **Note:** If DB has no qualifications yet (empty — first run), syncAll is a no-op. Qualification data enters via Plan 04 import flow, after which subsequent sync runs pick them up.
