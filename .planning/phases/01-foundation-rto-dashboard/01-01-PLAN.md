<objective>
Scaffold the pnpm + Turborepo monorepo and create the Prisma schema for all 10 core entities so that downstream API and frontend plans have a working `@repo/db` typed client to consume.

Purpose: Establish the foundation layer before any application code. The schema must include UUID PKs (gen_random_uuid() + @db.Uuid), soft-delete (`deleted_at`), timestamps (`created_at`, `updated_at`), partial unique indexes to handle re-create-after-soft-delete (RESEARCH.md Pitfall 4), and relation stubs for the 8 RTO workspace sub-entities so future phases can flesh them out without schema rewrites.

Output: Monorepo root files (pnpm-workspace.yaml, turbo.json, root package.json), the `packages/database` workspace with schema.prisma, generated Prisma client singleton, docker-compose for local PostgreSQL, .env.example documenting DATABASE_URL, and an applied initial migration with all tables present in the live database.
</objective>

    3. Create initial migration and apply: `pnpm --filter @repo/db exec prisma migrate dev --name init`.
       - This generates `packages/database/prisma/migrations/<timestamp>_init/migration.sql` and applies it to the database.

       Then re-apply with `pnpm --filter @repo/db exec prisma migrate deploy`.

    5. Generate the typed client: `pnpm --filter @repo/db exec prisma generate`. This writes to `packages/database/generated/client/`.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `.env` file containing `DATABASE_URL` and JWT secrets | mitigate | `.gitignore` contains `.env`; only `.env.example` (no real secrets) is committed (Task 1 acceptance) |
| T-01-02 | Tampering | Soft-delete bypass via duplicate ASQA codes | mitigate | Partial unique index `rtos_asqa_code_active_unique` enforces uniqueness only among non-deleted rows (Task 3) |
| T-01-03 | Repudiation | RTO record changes are untraceable | accept | Phase 7 introduces activity logs; Phase 1 only persists `created_at` / `updated_at`. Acceptable for internal <10-user tool. |
| T-01-04 | Information Disclosure | Schema exposes `users.password_hash` | mitigate | Field is hashed (bcryptjs in Plan 02); never returned by API serialisers (Plan 02 enforces). Schema column is named explicitly `password_hash` to prevent accidental serialisation. |
| T-01-05 | Denial of Service | Unbounded `findMany` on `rtos` | accept | Internal tool, <10 concurrent users, <1000 RTOs expected. Pagination deferred until measured need. |
| T-01-SC | Tampering | npm/pnpm package installs (turbo, prisma, @prisma/client) | mitigate | RESEARCH.md §Package Legitimacy Audit verified all packages [VERIFIED: npm registry] against official source repos; no `[ASSUMED]`/`[SUS]` packages. |
