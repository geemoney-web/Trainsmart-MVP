<!-- API contract is defined by Plan 02. Frontend codes against it but does NOT import API code directly. -->

Endpoints consumed (all under /api/v1, all require Authorization: Bearer <accessToken> EXCEPT login):
- POST /auth/login    body { email, password }                  -> { accessToken } + Set-Cookie refresh_token (httpOnly)
- POST /auth/refresh  Bearer refreshToken                       -> { accessToken } + Set-Cookie refresh_token (rotated)
- POST /auth/logout                                              -> 204, clears cookie
- GET  /rtos                                                     -> Rto[] each with `_count: { alerts, validations }`
- GET  /rtos/:id                                                 -> Rto with relations (qualifications[], trainers[], etc.)
- POST /rtos          body CreateRtoDto                          -> Rto (201)
- PATCH /rtos/:id     body UpdateRtoDto                          -> Rto
- DELETE /rtos/:id                                               -> Rto (soft-deleted; not removed)

Rto shape consumed by frontend:
  { id: string; name: string; asqa_code: string; operating_states: string[];
    contact_name?: string; contact_email?: string; contact_phone?: string;
    status_color: string;        // EXISTS but NEVER rendered in Phase 1 per D-09
    _count: { alerts: number; validations: number } }

CreateRtoDto shape (POST /rtos):
  { name: string (min 2); asqa_code: string (5-digit); operating_states: string[] (>=1);
    contact_name?: string; contact_email?: string (RFC email); contact_phone?: string }
</interfaces>
