<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API (presign) | Untrusted file metadata |
| Browser → S3 (direct PUT) | File bytes leave the browser only via presigned URL |
| Browser → API (persist) | TAS metadata persists via authenticated apiFetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Tampering | File type spoofing via extension | mitigate | Use file.type (MIME) for contentType field; backend rejects via PresignTasDto whitelist; accept attribute restricts UI selection |
| T-03-14 | Denial of Service | Oversized file selection | mitigate | Client-side check `file.size > 52_428_800` shows error before presign call; backend still enforces via `@Max` |
| T-03-15 | Information Disclosure | Presigned URL logged to console | mitigate | Never console.log presignedUrl; only error.message gets logged in catch blocks |
| T-03-16 | Spoofing | Upload via direct API call bypassing UI | accept | Backend already authenticates and validates DTOs; UI is not the security boundary |

</threat_model>
