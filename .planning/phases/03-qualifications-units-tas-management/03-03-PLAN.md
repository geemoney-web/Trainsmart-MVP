<objective>
Backend TAS lifecycle endpoints covering all six TAS requirements: upload (TAS-01), versioning with auto-archive (TAS-02, D-08), three-state status enum (TAS-03), append-only history (TAS-04), structured metadata (TAS-05), and RTO document library surfacing (TAS-06). Backend never handles file bytes — frontend uploads directly to S3 using a presigned URL issued from `POST /tas/presign`.

Purpose: Plan 05 (frontend TAS UI) cannot function without these endpoints. The auto-archive transaction is a critical correctness requirement — without it, two "Current" TAS records can exist for the same qualification.
Output: One controller with three routes, one service with three methods, an S3 client factory, two DTOs, a TasModule registered in AppModule, and Jest specs covering auto-archive correctness and Draft-does-not-archive.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API (presign request) | Untrusted file metadata (name, size, content-type) crosses here |
| Browser → S3 (direct PUT) | File bytes never traverse our backend (per CLAUDE.md) |
| API → Database (transaction) | Auto-archive must be atomic |
| RTO data isolation | TAS records are RTO-scoped; queries must filter by rto_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | POST /tas, POST /tas/presign, GET /tas/rtos/:rtoId | mitigate | Global AccessTokenGuard from AuthModule (Phase 1) inherited; verified by tas.controller.spec.ts |
| T-03-05 | Tampering | File upload to S3 | mitigate | Backend never handles bytes; presigned URL bound to specific Bucket+Key+ContentType; `expiresIn: 3600` (1 hour) limits replay window; PutObjectCommand uses fixed content type from PresignTasDto whitelist |
| T-03-06 | Tampering | Auto-archive race condition | mitigate | All status updates + record creation inside `prisma.$transaction` interactive callback so partial state cannot persist (D-08, RESEARCH.md Pattern 2) |
| T-03-07 | Information Disclosure | Cross-RTO TAS access via GET /tas/rtos/:rtoId | mitigate | findMany filtered by `rto_id` from URL param; future RBAC will verify user→RTO mapping (deferred per RESEARCH.md V4) |
| T-03-08 | Denial of Service | Oversized file upload | mitigate | `@Max(52_428_800)` on `fileSize` in both DTOs; presigned URL ContentLength implicitly bounded by S3 bucket limits |
| T-03-09 | Tampering | Arbitrary content-type upload | mitigate | PresignTasDto `@IsIn` whitelist limits to PDF + DOC + DOCX; content-type baked into presigned URL so S3 rejects mismatched uploads |

</threat_model>
